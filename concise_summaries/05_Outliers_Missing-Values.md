# 05 — Outliers & Missing Values

## Core Definitions & Concepts

- **Outlier (Grubbs, 1969):** An observation that appears to deviate markedly from other members of the sample.
- **Outlier (Barnett, 1994):** An observation that appears inconsistent with the remainder of the data.
- **Outlier (Hawkins, 1980):** An observation that deviates so much it arouses suspicion of being generated by a different mechanism.
- **Anomaly:** An observation that deviates considerably from a concept of normality; also called outlier or novelty.
- **Terminology distinction:**
  - _Anomaly_ — instance from a distinct distribution other than P+ (e.g., dog among cats).
  - _Outlier_ — rare/low-probability instance from P+ (e.g., wild cat).
  - _Novelty_ — new region/mode of an evolving, nonstationary P+ (e.g., new cat breed).
- **Types of anomalies:**
  1. **Point anomaly** — single deviating data point.
  2. **Contextual/conditional anomaly** — deviating only in a specific context (e.g., freezing temperature in the Amazon but not Antarctica).
  3. **Group/collective anomaly** — a set of related points that together are anomalous (e.g., coordinated network attacks).
  4. **Low-level sensory anomaly** — no high-level abstraction needed (e.g., pixel-level anomalies).
  5. **High-level semantic anomaly** — assumes latent representation Z and Y; normality conditioned on concepts (e.g., objects from non-normal classes).
- **Missingness mechanisms (Little & Rubin, 1987):**
  - **MCAR (Missing Completely at Random)** — no dependency on observed or missing data.
  - **MAR (Missing at Random)** — depends on observed data but not the missing value itself.
  - **NMAR (Not Missing at Random)** — depends on the missing value itself (selection bias).
- **Key assumptions for anomaly detection:**
  - **Concentration assumption** — normal data is bounded to a region; anomalies live in low-density regions.
  - **Manifold assumption** — normal data lives on a lower-dimensional manifold.
  - **Prototype assumption** — normal data can be characterized by a finite set of prototypes.

## Key Algorithms & Techniques

- **Density-based models** (generative): Gaussian distribution fitting, Kernel Density Estimation (KDE), Real NVP (normalizing flows).
- **One-class classification models** (discriminative): learn a decision boundary around normal data — hyper-sphere, ellipsoidal, OC-SVM, deep one-class methods.
- **Reconstruction-based models:** PCA, kernel PCA, autoencoders, variational autoencoders (VAEs) — reconstruction error signals anomalies.
- **Local Outlier Factor (LOF):** density-based score using k-nearest neighbours; points in less dense regions get higher anomaly scores.
- **Isolation Forest (iForest):** builds random decision trees; normal points require more splits to isolate.
- **Angle-Based Outlier Degree (ABOD):** uses variance of angles between data points; robust in high dimensions.
- **Grid-based subspace outlier detection:** partitions space into equi-depth grid cells; cells with sparsity coefficient S(C) < 0 are anomalous.
- **Depth-based techniques:** organize data into convex hull layers; points with low depth flagged as outliers.
- **Missing data imputation strategies:**
  - _Univariate imputation:_ global constant, most common value, mean/median.
  - _Multivariate imputation:_ predict missing values from other features via ML models.
  - _Ignore missing:_ use only methods that handle missing values (e.g., decision trees) or drop incomplete rows.

## Important Formulas & Diagrams

- **Anomaly set definition:** A = { x in X | p+(x) <= tau }, where tau >= 0 is a small probability threshold.
- **Reconstruction error (autoencoder):** s(x) = ||x - psi_d(psi_e(x))||^2 (L2 norm of original minus reconstructed).
- **ABOD score:** ABOD(p) = VAR( <x-p, y-p> / (||x-p||_2 * ||y-p||_2) ) — variance of cosine angles between p and all other points.
- **Grid-based sparsity coefficient:**
  S(C) = ( n(C) - n * (1/Phi)^k ) / sqrt( n * (1/Phi)^k * (1 - (1/Phi)^k) )
  where n(C) is object count in cell C, Phi cells per dimension, k dimensions.

## Learning Objectives

- Understand the generative nature of missing values and outliers (too little vs. too much data).
- Distinguish between anomalies (different generative process), outliers (rare from same process), and novelties (new modes of an evolving process).
- Recognize the three types of missingness (MCAR, MAR, NMAR) and their implications for imputation strategy.
- Understand the main anomaly detection settings: known distribution, unsupervised, semi-supervised, supervised.
- Be able to select appropriate algorithms (density-based, one-class classification, reconstruction, proximity-based, angle-based, grid-based) depending on data characteristics and assumptions.
- Appreciate that all methods make assumptions (concentration, manifold, prototype) and these must hold for valid results.
