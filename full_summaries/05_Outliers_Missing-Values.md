## 1. Recap — Foundational Concepts

### 1.1 Visualisation & Sampling
- **Barplots** visualise proportions (e.g., urban population in Austria 1980 vs. 2020); skewness can challenge some algorithms.
- **Boxplots** display quartiles and highlight outliers beyond whiskers (e.g., firearms per capita; top 5 outliers: USA, Falkland Islands, Yemen, New Caledonia, Serbia; Austria ranks 14th).
- **Scatter plots** (log/log scale) reveal multivariate relationships (e.g., biodiversity data from OurWorldInData).
- **Sampling assumption:** Data generation governed by an underlying distribution; observed points are independent samples. If the distribution is known, its parameters can be estimated (e.g., sampling from N(2,1) with varying instance counts — more samples yield a more faithful empirical distribution).

### 1.2 Kernel Density Estimate (KDE)
- Estimates the PDF from discrete observations using a kernel function. **Kernel type** (e.g., Gaussian, Epanechnikov) and **bandwidth** (smoothness) critically influence the estimate.

### 1.3 I.I.D. Assumption
- (1) **Independently** — no dependencies between data points; (2) **Identically distributed** — distribution unchanged during sampling. Foundation for many statistical/ML methods. Common for tabular data; not plausible for time series (autoregressive).

### 1.4 Law of Large Numbers (LLN) & Central Limit Theorem (CLT)
- **LLN:** Estimates converge to true values with more data; uncertainty reduces.
- **CLT:** Complex processes with many independent influences tend toward normality — encountering normal distributions is unsurprising.

## 2. Outliers — "A Case of Too Much Data"

### 2.1 Motivation
- More data generally yields better estimates (LLN).
- **Counterexample:** In small-data regimes, a single extreme point can skew estimates. Example: 10 points from N(0,1); one point at 6 (<0.005% probability) shifts the mean to -0.58.

### 2.2 Types by Target Objective
1. **Outliers as noise** — negatively impact results; we remove them (e.g., measurement errors).
2. **Outliers as the target** — pattern to detect; we find them (e.g., fraud detection).

### 2.3 Types by Analysis Perspective
1. **Data-driven** — low-probability area under assumed distribution (e.g., height <5% tail).
2. **Generation-based** — different underlying mechanism (e.g., human vs. Martian). May not appear extreme in feature space.

### 2.4 Formal Definitions
- **Grubbs (1969):** "An outlying observation...that appears to deviate markedly from other members of the sample."
- **Barnett (1994):** "An observation...which appears to be inconsistent with the remainder of that set of data."
- **Hawkins (1980):** "An observation which deviates so much...as to arouse suspicions that it was generated by a different mechanism."
- All are intuitive but vague — requiring operationalisation via thresholds and probability models.

### 2.5 Key Reference Works
- **Chandola, Banerjee & Kumar (2009):** "Anomaly Detection: A Survey" — ACM CSUR, 41(3), 1-58.
- **Ruff et al. (2021):** "A Unifying Review of Deep and Shallow Anomaly Detection" — Proc. IEEE, 109(5), 756-795.
- The lecture uses **anomaly** as the main term (following Ruff et al.).

### 2.6 Application Areas
1. Intrusion detection (cybersecurity)
2. Fraud detection (finance, insurance, healthcare, telecom) — UK: 111.8M GBP (2019)
3. Industrial fault/damage detection
4. Infrastructure monitoring
5. Stock market monitoring
6. Acoustic novelty detection
7. Medical diagnosis
8. Disease outbreak detection
9. Event detection (earth sciences)
10. Scientific discovery (chemistry, bioinformatics, genetics, physics, astronomy)

### 2.7 Challenges
- **Skewed datasets** — few anomaly examples; severe class imbalance.
- **Open class** — no ground truth; unbounded anomaly types.
- **High dimensionality** — complicates modelling and distance computations. Deep learning helps address this.

### 2.8 Formal Definition of an Anomaly
- X ⊆ R^D is the data space; P+ is the normal-behaviour distribution with density p+(x).
- An anomaly lies in a low-probability region under P+. **Anomaly set:** A = {x in X | p+(x) <= tau}, tau >= 0.

### 2.9 Five Types of Anomalies
1. **Point anomaly** — single deviating point (e.g., fraudulent transaction).
2. **Contextual anomaly** — deviating only in context (e.g., freezing temp in Amazon vs. Antarctica). P+ = P+(X | T). Time-series are prominent.
3. **Collective anomaly** — related points anomalous together (e.g., coordinated network attack).
4. **Low-level sensory anomaly** — detectable without abstraction (e.g., pixel-level).
5. **High-level semantic anomaly** — latent Z (continuous) and Y (discrete). Normality: p+(x | z, y).

### 2.10 Terminology
- **Anomaly:** Instance from a different distribution than P+ (e.g., dog among cats).
- **Outlier:** Rare instance *from* P+ (e.g., wildcat among domestic cats).
- **Novelty:** New region of evolving, nonstationary P+ (e.g., new cat breed). May warrant model updates.

### 2.11 Finding Normal Data
- Special cases: P+ known (physical laws, domain expertise).
- General case: P+ unknown, must be estimated from data.

### 2.12 The Anomaly Detection Task
- Learn a model predicting whether unseen x~ in X is an anomaly (x~ in A).

### 2.13 Approaches
**By output type:** (1) Probabilistic (threshold alpha); (2) Score-based (ranking); (3) Binary.  
**By methodology:** (4) Distance-based; (5) Domain-informed (heuristics).

### 2.14 Key Assumptions
1. **Concentration:** Normal data bounded to a region; anomaly detection = identify low-density region.
2. **Manifold:** Normal data on lower-dimensional manifold M embedded in X.
3. **Prototype:** Finite prototypical elements characterise normal data.

### 2.15 Experimental Settings
1. **Known distribution:** KDE and OC-SVM assume uniform anomaly distribution.
2. **Unsupervised:** Unlabelled data; may contain anomalies.
3. **Semi-supervised:** Partially labelled; only normal data available in special case.
4. **Supervised:** Fully labelled; requires all anomaly types — often unrealistic.

### 2.16 Running Example
- **"Big moon and small moon"** dataset: two interleaving half-moons (with/without noise) used to illustrate anomaly detection model behaviour under varying conditions.

### 2.17 Anomaly Detection Models

#### 2.17.1 Density-Based (Generative)
- Model entire P+: **Gaussian fit**, **KDE**, **Real NVP** (flow-based). Caveat: learns all density level sets, which may be overkill.

#### 2.17.2 One-Class Classification (Discriminative)
- Decision boundary between P+ and anomalies. **Hyper-sphere**, **OC-SVM**, **Deep one-class**. False alarm rate alpha.

#### 2.17.3 Reconstruction Models
- Encoder-decoder; score = reconstruction error. PCA, Kernel PCA (RBF = KDE), Autoencoders, VAEs.

#### 2.17.4 Prototype-Based
- Cluster data; each cluster = prototype. Requires suitable distance measure.

#### 2.17.5 Local Outlier Factor (LOF)
- Dense regions (k-NN) normal; less dense = higher score. Available in sklearn.

#### 2.17.6 Isolation Forest (iForest)
- Random splits; normal points need more splits to isolate. Available in sklearn.

#### 2.17.7 Depth-Based
- Convex hull layers; depth <= k are outliers.

#### 2.17.8 Angle-Based (ABOD)
- Outliers have smaller angle variance. ABOD(p) = VAR(cosine angles). Stable in high dimensions.

#### 2.17.9 Grid-Based Subspace
- Equi-depth grid; sparsity coefficient S(C) = (n(C) - n*(1/Phi)^k) / sqrt(n*(1/Phi)^k * (1 - (1/Phi)^k)). Outliers where S(C) < 0 (below expected count).

## 3. Missing Values — "A Case of Too Little Data"

### 3.1 Overview & Sources
- **Sources:** Faulty equipment, human errors, anonymisation, non-response.
- **Implications (Barnard & Meng, 1999):** Loss of efficiency, method incompatibility, bias.
- Missing values may indicate errors but not necessarily.

### 3.2 Strategies
1. Use methods handling missing values natively (e.g., decision trees).
2. Ignore missing data points (listwise deletion) — only for sufficiently large datasets.
3. Missing data imputation.

### 3.3 Imputation Methods
- **Univariate:** Global constant, mode, mean/median.
- **Multivariate:** Predict from other observed variables (regression, k-NN). Requires modelling feature dependencies.

### 3.4 Strategy Selection
- Understanding reasons for missingness is critical for choosing the correct approach.

### 3.5 Causes of Missing Data (Little & Rubin, 1987)

**MCAR — Missing Completely At Random:**
- Missingness depends on **neither observed data nor missing value** (e.g., random transmission errors).
- Only loss of efficiency; no bias; no pattern. Strategy: ignore or simple imputation.

**MAR — Missing At Random:**
- Missingness depends on **other observed features** (e.g., fast drivers do not report driving style).
- Potential bias if ignored; pattern exists. Strategy: multivariate prediction-based imputation.

**NMAR — Not Missing At Random:**
- Missingness depends on **the missing value itself** (e.g., high-accident drivers do not report accidents).
- Strong bias risk (selection bias). Simple imputation risky; specialised models needed.
- Fundamentally untestable from observed data alone.

### 3.6 Key Takeaway
- MCAR/MAR/NMAR assumptions **cannot be fully verified from observed data alone**.
- Test MCAR vs. MAR via correlation checks; MAR vs. NMAR requires domain knowledge.
- Sensitivity analyses help assess robustness to different missingness assumptions.

## 4. Conclusions

- Data quality issues — noise, outliers, missing values — are pervasive in real-world datasets.
- Methods addressing them always make assumptions about the data generation process.
- We must be aware of assumptions and validate them (or justify via domain knowledge).
- The choice of technique follows from problem nature and assumption plausibility.
- Often, the information needed to validate an assumption cannot be inferred from data alone — domain expertise is essential.
